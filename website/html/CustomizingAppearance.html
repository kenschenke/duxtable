Customizing Appearance

The appearance of <DuxTable> is controlled entirely using CSS. Every component uses a CSS class that can be overridden. See the DuxTable Basics topic for a quick example of how easily appearance can be changed.

If you are using webpack, it might be tempting to simply import the duxtable.css file. This is perfectly fine but will not allow you to override the CSS. This is because the CSS is applied by React during component creation. See the Getting Started topic for more information on using the <link> element to reference the CSS in a way that it can be overridden.

Below is every CSS class used by DuxTable. The classes are named in a way that will be familiar to anyone that has used the traditional HTML <table> element. However, DuxTable is built using CSS FlexBox, not HTML tables. This might be important for CSS styling.

Class Description
.duxtable-table This is the outer-most <div> inside which everything else for the table is rendered.
.duxtable-thead The entire header row, including column headers and group headers (if present).
.duxtable-tbody The <div> that contains all the rows, including status messages (fetching or empty), footers, and any extra rows for uniform table height.
.duxtable-scroll This class is applied when pagination is turned off and the body scrolls with fixed headers.
.duxtable-text-body This class is applied to all table cells.
.duxtable-text-muted This class is applied to status messages such as empty or fetching.
.duxtable-text-left This class is applied to cells that are left-aligned.
.duxtable-text-right This class is applied to cells that are right-aligned.
.duxtable-text-center This class is applied to cells that are center-aligned.
.duxtable-invisible This class is applied to table elements that should be rendered but kept invisible.
.duxtable-tr The class applied to the <div> containing the cells for a row. This is also applied to empty rows to maintain uniform table height.
.duxtable-tr:hover When the mouse is hovering over a table row.
