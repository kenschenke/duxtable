DuxTable Reference

Below is a list of every <DuxTable> component property as well as a list of the array properties for the columns property.

Property Default Value Description
bodyHeight (none) The body height in pixels. This is used to enable scrolling with fixed column headers. See the topic on Fixed-Header Scrolling tables for more information.
columns (none) This specifies the columns in the table. See the Basics topic for more information.
data (none) The row data. See the Basics topic for more information.
emptyMsg (none) Message that is displayed when there are no rows in the table
fetchingData false Set this property to true when fetching data from a remote source or other data store that might take a little while. The message in fetchingMsg is shown while this is true.
fetchingMsg "Fetching Data" This message is shown when fetchingData is true.
filterCallback (none) This function is called when the user types something in the search box in the table. It is passed the search string and a single row object from the array supplied in the data property. It should return true if the row should be included in the filtered table rows.
groupHeaders (none) Group headers. See the topic on Header Groups for more information.
mouseEnterRow (none) This function is called when the mouse enters a row. It is passed the row object from the array supplied in the data property.
mouseLeaveRow (none) This function is called when the mouse leaves a row. It is passed the row object from the array supplied in the data property.
name (required) The name of the table, such as "addresses"
pageSize 10 The page size for pagination. See the topic on Paginated tables for more information.
pagination true True if pagination is enabled. See the topic on Paginated tables for more information.
rowKey (none) The property in each row object that uniquely identifies each row. See the Basics topic for more information.
rowSelectionChanged (none) This function is called when the row selection changes. It is called with an array of the keys of the selected rows.
searchFocus false If true, the search box is given keyboard focus when the DuxTable component is initialized.
selectedRenderCallback (none) This function is called when a row is selected. See the topic on Selected Row Rendering for more information.
selectionMode "none" The row selection mode. One of "none", "single", or "multi".
showSearch true If true, a search box allows the user to filter table rows.
sortAscending true If true, rows are sorted in ascending order by the sortColumn. Note: this property is only observed during table initialization. If later changed, the new property value is ignored.
sortColumn -1 The column to initially sort rows (starting at 0). -1 means no sorting is initially done. Note: this property is only observed during table initialization. If later changed, the new property value is ignored.
striped false If true, alternating rows are given a slightly darked shade of the background color.

Table Columns

The columns are defined in the columns property to DuxTable as an array of JavaScript objects. The objects have several optional properties and only one required property: title. Here is a list of the properties followed by an example of a columns array in use.

Property Default Value Description
title (required) The title of the column shown in the column header
field (none) This is the field in the row object to render in the cell. In the above example, this could be Name or Email. If no custom rendering is necessary, this is the easiest way to display cell contents.
render (none) This property specifies a function to call to render the cell contents. The function is passed the object for the row from the array provided in the data property. It can return a string or a React component. See the topic on Custom Cell Rendering for more information.
align left This specifies the text alignment in the cell: "left", "center", or "right".
width (none) DuxTable automatically calculates column sizes based on the width of the table. This property specifies the width in pixels of the column. Columns without a specified width are sized equally. The user is allowed to resize columns by dragging the divider in the column header.
sortable true By setting this property to false, the user will not be allowed to click in the column header to sort the table by this column.
sortCallback (none) If present, this function will be called when the user clicks on the column header to sort the table by this column. See the topic on Row Sorting for more information.
footer (none) This specifies a value or React component to render as a footer for this column. See the topic on Footers for more information.
